NAME

parpd — a Proxy ARP Daemon

SYNOPSIS

parpd [-dfl] [-c file] [interface] [...]

DESCRIPTION

parpd is an implementation of the Proxy ARP Daemon described in RFC 1017. Upon receiving an ARP request, parpd looks up the target address in the configuration file, parpd.conf(5). If parpd finds a match, it replies with the associated hardware address or the hardware address of the interface on which the reply came in.

In normal operation, parpd forks a copy of itself and runs in the background. Errors are reported via syslog(3).

OPTIONS
-c file

Use this file for configuration instead of the default /etc/parpd.conf.

-d

Run in debug mode, with all the output to stderr. This option implies the -f option.

-f

Run in the foreground.

-l

Log all requests to syslog.

FILES
/etc/parpd.conf
