PARPD.CONF(5) File Formats Manual PARPD.CONF(5)

NAME

parpd.conf — Proxy ARP Daemon configuration file

DESCRIPTION

The parpd.conf file maps a range of inet addresses to be replied to by parpd(8). Lines consist of a command, an inet address and an option Ethernet address to respond with. Subsequent rules override prior rules. A ‘#’ character indicates the beginning of a comment; characters up to the end of the line are not interpreted by parpd.

Each line in parpd.conf has the format:

command [inet-address[/cidr][/netmask]][iface] [hardware-address]

Valid commands are proxy, halfproxy, ignore and interface. The halfproxy command is the same as proxy, but will ignore requests from an unspecified address. Using the interface command, you can specify an interface on which all subsequent commands are only used on. Inet addresses are specified in the conventional dot (“.”) notation with an optional cidr or netmask. Hardware addresses are expressed as six hexadecimal numbers separated by colons, e.g. "08:00:20:00:5a:bc".

attack is a special command which proxies the address after receiving two Gratuitous ARP relies - ie both source and destination addresses are the same. This is only useful for testing RFC 5227 implementations.
